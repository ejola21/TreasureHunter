# game_rule.md — 미션 디자이너 가이드

> **대상**: 미션을 직접 만드는 디자이너 / 크리에이터.
> **다루는 것**: 17개 아이템의 용도·장단점·설정·표시, 미션 메타 설정, 추천 디자인 패턴.
> **다루지 않는 것**: 코드·DB 스키마·서버 API → [game_rule_legacy.md](game_rule_legacy.md) (레거시 기술 사양서) / [item_design.md](item_design.md) / [item_design2.md](item_design2.md).

---

## 0. 한눈에 보기

```
미션 = Start (시작) + 필수/선택 아이템들 + End (종료)
        └──────────── 지도에 배치 ────────────┘
                +
   메타: 제목 · 설명 · 장소 · 언어 · 시간제한 · 공개여부 · 뱃지
```

**최소 조건**: Start 1개 + End 1개. 그 외 아이템은 자유.

**플레이 흐름**:
1. 플레이어가 Start 아이콘 근처 도달 → 미션 시작 (타이머 가동)
2. 지도 / AR 화면을 보며 다른 아이템들 수집
3. 모든 **필수(mandatory)** 아이템 획득 시 End 노출
4. End 도달 → 미션 클리어

## 1. 17개 아이템 카탈로그

### 🚩 미션 시작·종료 (필수)

#### **Start** — 시작 깃발 (초록 핀 + 체크무늬)

| 항목 | 내용 |
|---|---|
| 용도 | 미션 시작점. 플레이어가 이 위치에 들어와야 시간이 흐름 |
| 장점 | 자동 필수 처리, 안내문으로 동기부여 가능 |
| 단점 | 미션당 1개만 둘 수 있음 (2개 두면 검증 실패) |
| 설정 필드 | 표시방식, 발견거리, 안내문 |
| 표시 | 지도/AR 모두에 핀 |
| 디자이너 팁 | 시작 안내문은 "여기서 출발!" 같은 짧고 명확한 한 줄 |

#### **End** — 결승 깃발 (파란 핀 + 흰 깃발)

| 항목 | 내용 |
|---|---|
| 용도 | 미션 종료점. 필수 아이템 모두 수집 후에만 노출 |
| 장점 | 순서 강제 가능 (먼저 End 진입 못 하게 자동 차단) |
| 단점 | Start 와 너무 가까우면 미션 짧고 시시함 |
| 설정 필드 | 표시방식, 발견거리, 안내문 |
| 표시 | 지도에는 mandatory 아이템 ≤ 1 일 때만 등장 / AR 동일 |
| 디자이너 팁 | Start 에서 500m~2km 거리. 마지막에 들를 만한 의미 있는 장소 |

### 🎯 진행·학습 아이템

#### **Hint** — 힌트 (노란 핀 + 전구)

| 항목 | 내용 |
|---|---|
| 용도 | 안내 문구를 가르쳐 주는 가장 가벼운 아이템 |
| 장점 | 자유도 높음 — 필수/선택 토글, 미니게임 추가, 숨김 모두 가능 |
| 단점 | 안내문이 약하면 존재감 없음 — 잘 쓰면 분위기, 못 쓰면 의미 없는 점 |
| 설정 필드 | 필수여부, 표시방식, 발견거리, 미니게임, 안내문 |
| 표시 | 지도/AR 핀. showType 설정에 따라 한쪽 숨김 가능 |
| 디자이너 팁 | 안내문에 다음 아이템 단서, 가게 이름, 장소 의미 등을 넣자 |

#### **Quiz** — 퀴즈 (빨간 핀 + 물음표)

| 항목 | 내용 |
|---|---|
| 용도 | 정답을 맞춰야 획득되는 아이템 |
| 장점 | 학습 / 관광 정보 전달에 최적. 변형 N개 등록 시 매번 랜덤 출제 |
| 단점 | 정답 강제로 사용자가 막힐 수 있음 → Solution 아이템과 짝 권장 |
| 설정 필드 | 필수여부(자동 켜짐), 표시방식, 발견거리, **퀴즈 변형 1+** (질문/정답) |
| 표시 | 지도/AR 핀. 가까이 가면 퀴즈 창 |
| 디자이너 팁 | 변형 3개 이상 등록 시 재미 ↑. 정답은 짧고 명확하게 (대소문자 무관) |

#### **Run Start** — 타임어택 시작 (초록 핀 + 스톱워치/달리기)

| 항목 | 내용 |
|---|---|
| 용도 | 짧은 카운트다운 구간 시작. **Run End 와 한 쌍** |
| 장점 | 미션 안 미니챌린지로 긴장감 추가. 자동으로 페어 Run End 생성됨 |
| 단점 | 타임어택 실패 시 미션 자체 실패 — 난이도 균형 주의 |
| 설정 필드 | 필수여부(자동 켜짐), 표시방식, 발견거리, 안내문 |
| 표시 | 지도/AR 핀. 획득 즉시 카운트다운 시작 |
| 디자이너 팁 | 30~120초 권장. 빌더가 Run End 도 자동 추가하니 따로 안 만들어도 됨 |

#### **Run End** — 타임어택 종료 (초록 핀 + 스톱워치/깃발)

| 항목 | 내용 |
|---|---|
| 용도 | Run Start 의 짝꿍. 제한 시간 안에 획득해야 성공 |
| 장점 | 자동 페어링 — 별도 추가 안 해도 됨 |
| 단점 | 위치 옮기는 것 외에 직접 만질 일 거의 없음 |
| 설정 필드 | 필수여부(자동 켜짐), 표시방식, 발견거리, **제한 시간**, 거리(자동), 페어ID, 안내문 |
| 표시 | 지도/AR 핀. Run Start 획득 후에만 등장 의미 |
| 디자이너 팁 | Run Start 와의 거리가 자동 계산되니, 시간만 적절히 (도보 이동 가능 거리 + 여유 5~10초) |

### ⚠️ 위험·장애물 아이템

#### **Mine** — 지뢰 (빨간 핀 + 폭탄)

| 항목 | 내용 |
|---|---|
| 용도 | 반경 안에 들어가면 폭발, 마지막에 얻은 아이템 1개 분실 |
| 장점 | 동선에 긴장감. 좁은 길목·핵심 통로에 두면 효과 만점 |
| 단점 | 너무 많으면 좌절. 동선 막아버릴 수도 있음 |
| 설정 필드 | 필수여부(자동 꺼짐), **반경**(rangeAR) |
| 표시 | 지도에 **빨간 원** (반투명 채움) — 핀 아이콘 없음. 보통 가려져 있다가 Mine Radar 획득 시 등장 |
| 디자이너 팁 | 반경 20~50m 권장. 미션당 1~3개. 필수 동선 막지 말 것 |

#### **Dark** — 다크존 (검은 핀 + 과녁)

| 항목 | 내용 |
|---|---|
| 용도 | 지도 반경 안에서는 다른 아이템들이 숨겨짐 (보물찾기 분위기) |
| 장점 | 특정 구역을 "AR 탐색 전용" 으로 만들어 몰입감 ↑ |
| 단점 | Stealth Radar 없으면 안에 뭐가 있는지 전혀 모름 — UX 주의 |
| 설정 필드 | 필수여부(자동 꺼짐), **반경**(rangeAR) |
| 표시 | 지도에 **검은 원** (반투명) — 핀 아이콘 없음 |
| 디자이너 팁 | 반경 30~100m. 큰 광장·공원에 1개. Stealth Radar 함께 배치 권장 |

#### **Defense** — 방어막 (파란 핀 + 초록 방패)

| 항목 | 내용 |
|---|---|
| 용도 | 획득하면 다음 Mine 폭발 1회 무효화 |
| 장점 | Mine 미션의 안전망 — 난이도 완화 카드 |
| 단점 | 1회용. Mine 이 없는 미션엔 의미 없음 |
| 설정 필드 | 필수여부, 표시방식, 발견거리, 미니게임, 안내문 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | Mine 가 있는 미션에서만 의미. 동선 중간에 1개 |

### 💪 보조·레이더 아이템

#### **Solution** — 해답 부적 (파란 핀 + 학사모)

| 항목 | 내용 |
|---|---|
| 용도 | 획득하면 Quiz 1개의 정답을 자동으로 알려줌 |
| 장점 | Quiz 미션의 막힘 해소. 사용자 좌절 방지 |
| 단점 | 너무 쉽게 두면 Quiz 의 도전 의미 떨어짐 |
| 설정 필드 | 필수여부(자동 꺼짐), 발견거리, 미니게임 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | Quiz 가 3개 이상 있는 미션에 1개. 미니게임 추가로 획득 난이도 살짝 ↑ |

#### **Stealth Radar** — 스텔스 레이더 (보라 핀 + 도둑가면)

| 항목 | 내용 |
|---|---|
| 용도 | 획득 후 AR 화면에서 숨겨둔 아이템들이 보임 |
| 장점 | "AR 숨김(arOnly)" 아이템과 함께 쓰면 발견 미션 디자인 가능 |
| 단점 | 미션당 1개만 의미 있음 |
| 설정 필드 | 필수여부, 표시방식, 발견거리, 안내문 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | AR 숨김 아이템 3개 이상 있을 때 1개 배치 |

#### **Map Radar** — 지도 레이더 (보라 핀 + 과녁)

| 항목 | 내용 |
|---|---|
| 용도 | 획득 후 지도 화면에서 숨겨둔 아이템들이 보임 |
| 장점 | "MAP 숨김(mapOnly)" 아이템과 함께 쓰면 효과적 |
| 단점 | 미션당 1개만 |
| 설정 필드 | 필수여부, 표시방식, 발견거리, 안내문 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | MAP 숨김 아이템 3개 이상 있을 때 1개 |

#### **Mine Radar** — 지뢰 레이더 (보라 핀 + 폭탄과녁)

| 항목 | 내용 |
|---|---|
| 용도 | 획득 후 지도에 지뢰 빨간 원이 표시됨 |
| 장점 | Mine 가 많은 코스의 생존율을 크게 올림 |
| 단점 | Mine 없는 미션엔 무용 |
| 설정 필드 | 필수여부, 표시방식, 발견거리, 안내문 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | Mine 가 2개 이상이면 1개 두는 걸 강력 추천 |

#### **All Radar** — 모든 레이더 (보라 핀 + 큰 별 + 신호점)

| 항목 | 내용 |
|---|---|
| 용도 | 위 3개 레이더 효과를 한 번에 |
| 장점 | 미션 마지막 보상 / 상점 사용 등 특별 보상으로 |
| 단점 | 너무 일찍 주면 다른 레이더 의미 사라짐. 사용 빈도 낮음 |
| 설정 필드 | 필수여부, 표시방식, 발견거리, 안내문 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | 보통 미션엔 안 쓰는 편이 깔끔 |

### 🎁 보너스·재미 아이템

#### **Gambling** — 랜덤 박스 (주황 핀 + 보물상자)

| 항목 | 내용 |
|---|---|
| 용도 | 획득하면 다른 미획득 아이템 1개를 함께 자동 획득 (행운) |
| 장점 | 진행 단축의 깜짝 보상. 분위기 띄움 |
| 단점 | 랜덤이라 예측 어려움. 너무 많으면 미션 너무 쉬워짐 |
| 설정 필드 | 필수여부, 표시방식, 발견거리, 미니게임, 안내문 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | 미션당 1~2개. 미니게임 (흔들기) 추가로 획득 의식 부여 |

#### **Coupon** — 쿠폰 (핑크 핀 + % 마크)

| 항목 | 내용 |
|---|---|
| 용도 | 획득하면 쿠폰 코드/안내문이 알림으로 표시 |
| 장점 | 실제 가맹점 할인·이벤트 연동 가능 (제휴 미션) |
| 단점 | 안내문 안 적으면 텅 빈 알림. 잘못된 쿠폰코드 노출 위험 |
| 설정 필드 | 필수여부, 표시방식, 발견거리, 미니게임, 안내문 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | 안내문에 실제 쿠폰 코드 + 사용 안내. 예: `"이마트 5000원 할인 ABCD-1234 / 3월까지 사용"` |

#### **Store** — 상점 (황금 핀 + 쇼핑백) — 🚧 준비 중

| 항목 | 내용 |
|---|---|
| 용도 | 향후 결제/포인트 화면 진입점 (현재 미구현) |
| 장점 | — |
| 단점 | 현재 배치해도 게임 중 알림 안 뜸. **현재 사용 금지 권장** |
| 설정 필드 | 필수여부(자동 꺼짐), 발견거리 |
| 표시 | 지도/AR 핀 |
| 디자이너 팁 | 정식 출시 전까지 미사용 |

## 2. 5개 입력 필드 — 통합 매트릭스

빌더의 아이템 상세 화면에는 다음 5개 필드가 있습니다 ([item_design2.md](item_design2.md) 참조):

| 필드 | 의미 | 값 범위 |
|---|---|---|
| **필수 여부** | 미션 클리어에 꼭 필요? | ON/OFF (자동/사용자) |
| **표시 방식** | 어디에 보일지 | 숨김없음 / AR 숨김 / MAP 숨김 |
| **발견 거리** | 발견되는 유효 반경 (m) | 5~500, step 5 |
| **미니게임** | 획득 시 동작 | 0=없음 / 1=흔들기 ✅ / 2=터치 🚧 / 3=랜덤 🚧 |
| **안내 문구** | 획득 시 알림 본문 | 자유 텍스트 |

**아이템별 노출 필드**:

| 아이템 | 필수 | 표시 | 거리 | 미니게임 | 안내문 | 특수 |
|---|---|---|---|---|---|---|
| Start | 자동 ON | ✅ | ✅ | — | ✅ | — |
| End | 자동 ON | ✅ | ✅ | — | ✅ | — |
| Hint | ✅ | ✅ | ✅ | ✅ | ✅ | — |
| Quiz | 자동 ON | ✅ | ✅ | — | — | **변형 1+** |
| Run Start | 자동 ON | ✅ | ✅ | — | ✅ | 페어ID |
| Run End | 자동 ON | ✅ | ✅ | — | ✅ | **제한시간** + 거리(자동) + 페어ID |
| Mine | 자동 OFF | — | ✅ | — | — | 폭발반경 강조 |
| Dark | 자동 OFF | — | ✅ | — | — | 다크존반경 강조 |
| Defense | ✅ | ✅ | ✅ | ✅ | ✅ | — |
| Solution | 자동 OFF | — | ✅ | ✅ | — | — |
| Gambling | ✅ | ✅ | ✅ | ✅ | ✅ | — |
| 각 Radar | ✅ | ✅ | ✅ | — | ✅ | — |
| Coupon | ✅ | ✅ | ✅ | ✅ | ✅ | — |
| Store | 자동 OFF | — | ✅ | — | — | — |

## 3. 미션 메타 정보 설정

빌더의 첫 화면 (MissionSetupView):

| 항목 | 의미 | 필수? | 비고 |
|---|---|---|---|
| **제목** | 미션 카드/리스트에 표시 | ✅ | 짧고 매력적으로 |
| **설명** | 상세 화면 본문 | ✅ | 미션 소개·테마 |
| **장소** | 지역 라벨 | ⛔(자동) | Start 배치 시 reverseGeocode 로 자동 채움 |
| **언어** | 미션 대상 언어 | ✅ | 기본 `ko` |
| **시간 제한** | 0=무제한, >0=카운트다운 | — | HH:MM:SS 입력 |
| **공개 상태** | 비공개(작성중) / 공개(Missions 탭 노출) | — | 토글 |
| **뱃지 이미지** | 미션 카드 썸네일 | — | 1:1 크롭 + JPEG 0.85 자동 처리 |
| **가상 모드** | Real / Virtual 양쪽 / Real 만 | — | Virtual 켜면 어디서나 플레이 가능 |

## 4. 추천 미션 디자인 패턴

### A. 입문형 — 4 아이템 튜토리얼 (tutorial001 스타일)
```
Start  →  Hint  →  Quiz  →  End
```
- 모두 mandatory
- Quiz 변형 1개만 (간단한 정답)
- 안내문에 게임 룰 설명

### B. 표준형 — Quiz 중심 학습 (5~10 아이템)
```
Start → Quiz×3 → Solution×1 → End
                       ↑
              (Quiz 막힘 안전망)
```
- Quiz 변형 3개씩 → 매번 다른 문제
- Solution 1개로 좌절 방지

### C. 액션형 — 긴장감 추구 (지뢰 미션)
```
Start → Hint × 2 → Mine × 2 → Mine Radar → Defense → End
              ↑                      ↑           ↑
           코스 안내            안전망         보험
```
- Mine 반경 30m × 2
- Mine Radar 로 보이게 + Defense 로 1회 보호

### D. 탐험형 — 보물찾기 (다크존 미션)
```
Start → Dark(반경 80m) → Hint(AR 숨김) × 3 → Stealth Radar → End
                                ↑                    ↑
                          AR 에서만 찾기      AR 라더로 표시
```
- 큰 광장에 Dark 1개 + 그 안에 AR 숨김 Hint 3개
- Stealth Radar 미획득 시 거의 못 찾음

### E. 타임어택형 (러닝)
```
Start → Hint × 2 → Run Start(60초) → Run End → End
                          ↑              ↑
                        시간 시작     도착해야 성공
```
- Run Start 위치까지 충분한 워밍업 (2~3개 Hint)
- 제한 시간은 도보 이동 시간 + 여유 10초

### F. 보너스형 (이벤트/쿠폰)
```
Start → Quiz × 2 → Gambling → Coupon → End
                       ↑          ↑
                  랜덤 보상   실제 할인 코드
```
- 가맹점 제휴 시 Coupon 안내문에 실제 쿠폰 코드
- Gambling 으로 깜짝 단축 효과

## 5. 검증 체크리스트 — 저장 전 확인

빌더가 자동 검증하지만 디자이너도 확인 권장:

- [ ] **Start 정확히 1개**
- [ ] **End 정확히 1개**
- [ ] mandatory 아이템 1개 이상 (Start 자기 자신 제외)
- [ ] Start ↔ End 거리 200m 이상 (너무 가까우면 무의미)
- [ ] Quiz 아이템마다 변형 1개 이상 + 정답 입력
- [ ] Run Start 가 있으면 Run End 자동 페어 확인
- [ ] Mine 의 반경이 다른 아이템과 100% 중첩되지 않는지
- [ ] Dark 반경 안에 Stealth Radar 가 노출되는지 (Map Radar 만으론 안 보임)
- [ ] 모든 안내문이 비어 있지 않은지 (선택 사항이지만 비어 있으면 알림이 빈 칸)
- [ ] 뱃지 이미지 첨부 (없으면 placeholder)
- [ ] 제목 + 설명 비어 있지 않음

## 6. 아이템 색상 가이드 (지도/AR 핀 식별용)

| 색상 | 아이템 | 의미 |
|---|---|---|
| **초록** | Start, Run Start, Run End | 시작·시간 |
| **파랑** | End, Defense, Solution | 종료·보조 |
| **빨강** | Mine, Quiz | 위험·도전 |
| **노랑** | Hint | 안내·도움 |
| **주황** | Gambling | 보너스 |
| **보라** | Map/Mine/Stealth/All Radar | 정보 (4종) |
| **검정** | Dark | 숨김 영역 |
| **핑크** | Coupon | 보상 |
| **황금** | Store | 상점 |

→ 색상으로 종류 직관적 식별 가능. 디자이너는 핀 색상으로 균형 잡힌 미션 구성을 시각적으로 검토 가능.
